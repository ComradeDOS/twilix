Twilix.Version Documentation¶

Module implements jabber:iq:version feature. (XEP-0092)

You can use this both to get versions of other entities and to represent your version to others.

class twilix.version.ClientVersion(dispatcher, client_name=None, client_version=None, client_os=None)¶

Version service class. Used to represent your own version to others and to ask others versions.

Parameters:
  • dispatcher – dispatcher instance to be used with the service.
  • client_name – client name to be represented as yours.
  • client_version – client version to be represented as yours.
  • client_os – client OS to be represented as yours.
getVersion(jid, from_=None)¶

Makes get version query to some JID.

Parameters:
  • jid – reciever for get version query
  • from – sender for get version query (if differs from myjid)
Returns:

deferred object which waits for result stanza with version info (or error stanza) from query’s target
init(disco=None, handlers=None)¶

Registers handlers and adds version feature in disco. Needs to be called if you want to represent your version to others.

Parameters:handlers – extra handlers to handle versions for JIDs that

are different from myjid.
