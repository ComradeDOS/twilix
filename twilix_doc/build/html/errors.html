Errors Documentation¶

Describe all expected classes of errors.

class twilix.errors.AppError(**kwargs)¶

Extends class Error. Contains String Node for special application condition as described in rfc 3920.

class twilix.errors.Condition(**kwargs)¶

Extends VElement from twilix.base. Contains condition value.

class twilix.errors.ConditionNode(*args, **kwargs)¶

Extends ElementNode from twilix.fields. Contains Condition element.

class twilix.errors.Error(**kwargs)¶

Extends VElement from twilix.base. Contains string attribute type, condition node and text node as described in rfc 3920.

clean_type_(value)¶

Cut off errors with wrong type. :returns: value if it has correct type. :raises: ElementParseError if value has a wrong type.

exception twilix.errors.ExceptionWithAppCondition(app_condition, *args, **kwargs)¶

Extends class ExceptionWithType. Used to describe undefined exception with special application description of error. Contains content field - Application Error element.

exception twilix.errors.ExceptionWithContent(*args, **kwargs)¶

Extends class ExceptionWithType. Contains content field - Error element.

exception twilix.errors.ExceptionWithType(type=None, reason=None, *args, **kwargs)¶

Extends class Exception. Define type and reason fields. Base class for special exceptions.

exception twilix.errors.UndefinedConditionException(app_condition, *args, **kwargs)¶

Describe undefined condition.

twilix.errors.condition_to_name(condition)¶

Bring condition to CapWords style. Used to define standart exception from rfc 3920. :returns: condition in CapWords style.

twilix.errors.exception_by_condition(condition)¶

Return exception appropriate to condition.
