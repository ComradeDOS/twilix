Twilix.Roster Documentation¶

Module describes client roster iteraction.

Roster handles a contact list, allows to add or remove contacts. Also roster handles contacts statuses and sends signals where some event happens.

class twilix.roster.Roster(dispatcher, mypresence=None)¶

Class describes interaction dispatcher with roster.

Signals described here:

roster_got: fired when roster is ready to use.

roster_item_added: fired when roster has a new item.

roster_item_removed: fired when roster has a removed item.

contact_available: fired when some contact is appeared online.

contact_unavailable: fired when some contact goes offline.

resource_available: the same as for contact_available but for separate resource not for whole contact.

resource_unavailable: the same as for resource_available but when contact goes offline.

resource_changed_status: fired when some resource changed it’s status information.

Parameters:
  • dispatcher – dispatcher for send/receive stanzas
  • mypresence – set initial presence to mypresence.

class twilix.roster.RosterPresence(*args, **kwargs)¶
