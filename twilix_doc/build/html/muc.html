Twilix.MUC¶

class twilix.muc.MultiChat(dispatcher)¶

Class implements multi chat user extension.

enter_room(presence, room_jid, nickname)¶

Sends ‘available’-type presence which allows client to enter the room. Fails if user already in room.

Parameters:
  • room_jid – JID of room-conference
  • nickname – string-type client’s nickname in conference
  • status – string-type client’s status message
get_list(room_jid, affiliation=None, role=None)¶

Sends query for receiving list of all users with fixed value of role or affiliation in room. Uses mostly of chat administrators.

Parameters:
  • room_jid – JID of room-conference
  • role – value of role for users will be filtered (default None)
  • affiliation – value of affiliation for users will be filtered (default None)
init()¶

Makes some initialization actions

leave_room(presence, room_jid, nickname)¶

Sends ‘unavailable’-type presence which leaves client from the room. Fails if user not in room.

Parameters:
  • room_jid – JID of room-conference
  • nickname – string-type client’s nickname in conference
set_affiliation(room_jid, jid, affiliation, reason=None)¶

Sends query for changing user’s affiliation to room jid. Uses mostly of chat administrators.

Parameters:
  • room_jid – JID of room-conference
  • jid – JID of user
  • affiliation – value of new user’s affiliation
  • reason – string-type argument with reason of affiliation changing (default None)
set_role(room_jid, nick, role, reason=None)¶

Sends query for changing user’s role in chat to room jid. Uses mostly of chat administrators.

Parameters:
  • room_jid – JID of room-conference
  • nick – string-type user’s nickname
  • role – value of new user’s role
  • reason – string-type argument with reason of role changing (default None)

anyHandler()¶

Changes list of info about users in chat. There is addition in roster for ‘available’-type presences and deletion from roster for ‘unavailable’-type. Also sends ‘user_available’/’user_unavailbale’ signals for dispatcher (from pydispatch module).
